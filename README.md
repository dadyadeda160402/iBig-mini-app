# Telegram Mini App для iBig (сервисный центр)

Проект в одной папке: `app.py` (Flask + Telegram bot) + `index.html` (весь фронтенд) + `requirements.txt`.
База данных `database.db` создаётся автоматически при первом запуске.

## Что умеет мини-апп

1. `Запись на ремонт` — форма (имя, телефон, тип устройства, описание, удобное время) отправляет заявку, а администратору приходит уведомление в Telegram.
2. `Прайс-лист` — таблица услуг с ценами по категориям.
3. `Задать вопрос` — вопрос уходит администратору в Telegram, администратор может ответить прямо в чате, ответ придёт клиенту.
4. `Статус ремонта` — ввод номера заявки, получение статуса.
5. `Контакты` — адрес, телефон, режим работы и карта.

## 1) Подготовка Telegram бота

### 1.1 Создайте бота через BotFather

1. Откройте BotFather: `@BotFather`
2. Команда: `/start`
3. Команда: `/newbot`
4. Укажите имя бота
5. После создания BotFather выдаст **BOT TOKEN** (это длинная строка вида `123456:ABC-...`)

Эту строку нельзя публиковать.

### 1.2 Узнайте свой `ADMIN_CHAT_ID`

Вам нужен ваш Telegram ID (обычно это ваш `chat_id` в личке с ботом).

Варианты:

1. Откройте сервис `@userinfobot` или `@getidsbot` в Telegram, нажмите `/start`.
2. Либо создайте одноразовый бот и выведите `chat_id` в ответ на сообщение.

Дальше используйте числовое значение как `ADMIN_CHAT_ID`.

## 3) Локальный запуск (для проверки)

1. Создайте `venv` и установите зависимости:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```
2. Поставьте переменные окружения (лучше через `.env`, но на практике можно и напрямую):
   ```bash
   export BOT_TOKEN="ВАШ_BOT_TOKEN"
   export ADMIN_CHAT_ID="ВАШ_ЦИФРОВОЙ_ID"
   ```
   `WEBHOOK_URL` не обязательно для локального запуска.
3. Запустите сервер:
   ```bash
   python3 app.py
   ```
4. Откройте `http://localhost:8000/`

> При локальном запуске Telegram Webhook не будет настроен, поэтому тестировать именно “ответ админа в Telegram” лучше на Railway после деплоя.

## 4) Деплой на Railway (бесплатный, через GitHub)

### 4.1 Создайте репозиторий на GitHub

1. Создайте новый репозиторий (например, `iBig-mini-app`)
2. Загрузите в него файлы текущей папки (`app.py`, `index.html`, `requirements.txt`, `README.md`)
3. Запушьте в `main` (или `master`)

### 4.2 Подключите к Railway

1. Перейдите в Railway: [https://railway.app](https://railway.app)
2. Нажмите **New Project**
3. Выберите **Deploy from GitHub**
4. Выберите репозиторий и ветку
5. Назовите сервис (например, `iBig-mini-app`)
6. Дальше Railway подберёт запуск, а мы зададим команду старта.

### 4.3 Настройте переменные окружения в Railway

В Railway откройте раздел **Variables** и добавьте:

1. `BOT_TOKEN` = ваш токен из BotFather
2. `ADMIN_CHAT_ID` = ваш Telegram ID (число)
3. `WEBHOOK_URL` = базовый публичный URL вашего сервиса на Railway (например: `https://your-app-production.up.railway.app`)

Важно: `WEBHOOK_URL` должен быть публичным и начинаться с `https://`.

### 4.4 Команда старта (Start Command)

В репозитории есть файл **`Procfile`** — Railway обычно подхватывает его сам и запускает `gunicorn` без ручной настройки.

Если вручную поле **Start Command** пустое или неправильное, укажите:

```bash
gunicorn app:app --bind 0.0.0.0:$PORT --workers 1 --timeout 120
```

Почему `--workers 1`: чтобы надёжно и без лишних копий приложения обработать webhook.

### 4.5 Дождитесь деплоя и убедитесь, что сервер доступен

После деплоя Railway выдаст публичный URL. Откройте его в браузере, чтобы убедиться что фронтенд загружается.

После запуска `app.py` автоматически установит Telegram webhook по адресу:
`WEBHOOK_URL + /telegram-webhook`

## 5) Подключение Mini App в BotFather (важно)

Чтобы Telegram Mini App открывался по кнопке в вашем боте:

1. Откройте BotFather и выберите ваш бот
2. Команда: `Edit Bot`
3. Найдите пункт: `Mini App`
4. Укажите URL вашего мини-аппа (это тот URL Railway, который открывает `index.html`, например `https://your-app-production.up.railway.app/`)

После этого кнопка в Telegram будет открывать ваш mini-app.

## 6) Как администратору отвечать на вопросы

1. Администратор получает уведомление в Telegram.
2. В уведомлении есть кнопка **Ответить**.
3. Админ отправляет ответ одним сообщением.
4. Ответ уходит клиенту автоматически.

Команда отмены: `cancel` (или `/cancel`, или слово `отмена`).

## 7) Как администратору менять статус ремонта

По умолчанию новые заявки имеют статус `new` (“Принята в работу”).

Администратор может менять статус командой в Telegram:

- `/setstatus <номер_заявки> <status>`
- `status` принимает значения: `new`, `in_progress`, `ready`, `done`

Пример:

`/setstatus iBig-20260323-A1B2C3 in_progress`

## Частые проблемы

### Сайт по ссылке Railway не открывается (пусто, 502, «Application failed»)

1. **Проверьте логи деплоя** в Railway: **Deployments → последний деплой → View Logs**. Если там ошибка импорта или падение процесса — приложение не поднимается.
2. **Обязательно задайте переменные** `BOT_TOKEN` и `ADMIN_CHAT_ID` в **Variables** (без кавычек и пробелов вокруг значений).  
   `ADMIN_CHAT_ID` — только цифры, например `123456789`.
3. После деплоя откройте в браузере:  
   `https://ваш-домен.railway.app/health`  
   Должен ответить JSON с `"ok": true`. Если `"config_ok": false` — в ответе будет подсказка, что не задано в Variables.
4. Убедитесь, что сервис **публичный** (есть публичный домен / генерация URL в Railway).  
5. Убедитесь, что **Start Command** в итоге запускает `gunicorn` (см. раздел 4.4 и файл `Procfile`).

### Другие ошибки

1. `Неверный initData Telegram`
   - чаще всего значит, что на фронтенде отправляется не то значение `initData`.
2. Админ не может отправить ответ клиенту
   - возможно, бот не может написать пользователю (бот не инициировал чат). Обычно это работает, если мини-апп открывается пользователем внутри диалога с нужным ботом.
